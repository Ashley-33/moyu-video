# Chrome Web Store 上架材料 & 说明

> ⚠️ **先读「审核风险」一节**。这个扩展的核心功能大概率通不过 Google 审核，本文件如实标注原因——能提高过审率的只有**改功能**，不是改文案。

## 一、上架前检查清单

- [x] 图标 16 / 48 / 128 PNG（`icons/`，已生成）
- [x] manifest 已声明 `icons` + `action.default_icon`
- [ ] **截图** ≥1 张，1280×800 或 640×400（需你在浏览器里实拍，见下）
- [ ] **隐私政策 URL**（把下面「隐私政策」全文托管到一个公开网址，如 GitHub Pages / Gist）
- [ ] 开发者账号（一次性 $5）
- [ ] 打包 zip（`./pack.sh`）

### 截图怎么拍
在一个普通网页上盖一个视频，全屏截图后裁成 1280×800。建议 2–3 张：覆盖播放中、popup 控制面板、hover 控制条。**注意别截到 B 站/小红书的 logo 或版权内容**（否则更容易被判侵权）。

## 三、隐私政策（把这段托管成公开网址，填到商店「隐私」栏）

> **摸鱼视频 隐私政策**
>
> 本扩展**不收集、不存储、不上传**任何个人数据，也不含任何统计/追踪代码。
>
> - **本地存储**：仅用 `chrome.storage` 保存你的音量、透明度等偏好，只存在你本机。
> - **主机权限**：仅用于在你主动粘贴链接时，解析对应站点的视频地址并播放；不在后台读取或发送你的浏览数据。
> - **declarativeNetRequest**：仅用于给视频流请求补必要的请求头以完成播放，不记录、不修改其他任何请求。
>
> 联系方式：<你的邮箱>

## 四、⚠️ 审核风险（诚实评估）

按被拒概率从高到低：

1. **知识产权（最致命）**：扩展会解析 B 站 `playurl` / 小红书 `masterUrl` 的**视频直链**、绕过官方播放器。这违反 Chrome Web Store 的 **IP 政策**，也违反 B 站/小红书的服务条款。审核会看代码，**改描述没用**。→ 唯一解法：删掉直链解析，只做合规内容。
2. **权限用途**：多站点 host 权限 + `declarativeNetRequest` 改 `Referer`。审核会追问「改请求头做什么」，改 Referer 绕防盗链容易被判可疑。
3. **欺骗性行为**：老板键快速隐藏、透明覆盖、「不留品牌痕迹」。若结合宣传语，可能被判 deceptive。文案写成中性「画中画播放器」能降低这一条的争议，但**功能仍在**。

**结论**：真心想上架，唯一有效的路是**移除盗链解析**（改成只嵌自有/授权内容或用户自己的视频），其余都是治标。是否值得，你定。

## 五、打包 & 提交

```bash
./pack.sh          # 生成 moyu-video-<版本>.zip（自动排除 README/STORE/pack 等）
```

然后到 [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole)：
新建条目 → 上传 zip → 填商店信息/隐私政策/截图 → 提交审核（几天到几周）。
